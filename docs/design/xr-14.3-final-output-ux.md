# XR-14.3 — Biopolitical Final Judgment + Lower-Harm Alternative UX

## Purpose

XR-14.3 closes the question-driven biopolitical workflow with a user-friendly final-output gate. The goal is not to make the model more decorative. The goal is to prevent premature certainty and make the final answer useful to a normal user.

This milestone converts the final layer of the Jarbou3i Life-Governance Analysis Engine into a visible UX contract: evidence-capped judgment, dominant mechanism, strongest evidence, weakest point, strongest objection, conditional prediction, watch indicators, disconfirmation conditions, and lower-harm alternative.

## Product rule

No final judgment without lower-harm alternative.

The engine can critique care, risk, classification, surveillance, exposure, and market capture, but the output should also produce a better option when possible.

## User-facing output checks

1. Evidence-capped judgment — Final confidence must not exceed ECS or the confidence caps.
2. Dominant mechanism — The result must name whether the main mechanism is classification, measurement, normalization, securitization, market capture, exposure, self-regulation, or institutional residue.
3. Strongest evidence and weakest point — The user must see both support and uncertainty.
4. Conditional prediction — The model must include probability, time horizon, and watch indicators.
5. Disconfirmation conditions — The output must say what would weaken or disprove the claim.
6. Lower-harm alternative — The model must propose a lower-harm alternative, trade-off, and residual risk.

## Why this matters

Without this gate, the biopolitical mode can become theory-heavy and overly confident. With this gate, the final report becomes probabilistic, auditable, practically useful, and less confusing.

## UX placement

- Simple Mode keeps the question-first workflow readable.
- Expert Mode exposes the full final judgment cockpit.
- Evidence Mode still precedes confidence.
- Final Mode turns analysis into decision-grade output.
