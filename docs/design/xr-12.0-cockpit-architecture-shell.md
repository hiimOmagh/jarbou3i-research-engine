# XR-12.0 — Cockpit Architecture Shell

## Decision

XR-12.0 starts the migration from a polished prompt workbench toward the new expert cockpit product architecture. RC-1 remains blocked until the migrated cockpit path is visually and structurally acceptable.

The target architecture is deliberately split:

- **Simple Mode** remains the guided entry workflow for normal users.
- **Expert Mode** gains a dark intelligence cockpit for imported analysis.
- **Export** remains the editorial report surface.

This milestone introduces the shell only. It does not replace the current guided flow, does not change import/export contracts, and does not alter provider, OAuth, backend, storage, or schema behavior.

## Product model

The cockpit shell is based on the workflow:

```text
Source → Structure → Brief
```

This model is stronger than the previous long-form review surface because it maps directly to how the product should be understood:

1. **Source panel** — what was imported, what lens/model/context is loaded, and whether source/evidence material exists.
2. **Structure panel** — how the answer was parsed into model layers, how many items exist, and where flags or contradictions appear.
3. **Brief panel** — the editorial preview of the analysis, top findings, confidence cues, and export path.

## Implementation scope

XR-12.0 adds:

- `#cockpitShellPanel`
- `data-xr12-cockpit-shell="source-structure-brief"`
- `data-xr12-cockpit-grid="staged-desktop-panels"`
- Source panel
- Structure panel
- Brief panel
- Empty state when no analysis is loaded
- Loaded state when `state.analysis` exists
- Review/export jump buttons through existing review tabs
- Mobile staged panels via one-column stacking

## Contract preservation

XR-12.0 preserves:

- runtime IDs used by existing tests
- prompt generation behavior
- JSON import behavior
- review dashboard behavior
- export contracts
- RTL/mobile no-overflow behavior
- XR-8 evidence matrix
- prompt sample visibility
- advanced controls visibility

## Simple Mode and Expert Mode responsibilities

### Simple Mode

Simple Mode should remain:

```text
Topic → Lens → Copy Prompt → Paste Answer → Review
```

It is the guided path and must not be overloaded with the full cockpit density.

### Expert Mode

Expert Mode should increasingly become:

```text
Source → Structure → Brief → Evidence → Export
```

The new dark intelligence cockpit is the correct target for Expert Mode and post-import review.

## Mobile model

The desktop shell uses three columns, but mobile cannot copy that layout directly. On mobile, the cockpit becomes staged panels:

```text
Source
Structure
Brief
```

The first version uses one-column stacking. Later milestones can add explicit panel tabs or a segmented mobile cockpit rail.

## Acceptance criteria

XR-12.0 is acceptable only if:

- Simple Mode still works.
- Expert Mode exposes a dark intelligence cockpit shell.
- Source panel, Structure panel, and Brief panel exist.
- Empty state is clear when no analysis is loaded.
- Imported analysis populates cockpit panels.
- Existing review and export flows still work.
- Browser core passes.
- Hosted evidence passes.
- Stable readiness passes.
- No-browser passes.

## Known limitations

This is an architecture shell, not the final cockpit rebuild. It does not yet fully migrate the systems map, evidence pressure board, contradiction module, scenario module, or export preview into the new cockpit design language.

Those should be handled in later milestones.
