# XR-11.4 — Mobile-First Command Flow Compression

## Status

Documentation and CSS implementation package for the post-audit visual rebuild track.

RC-1 remains blocked until the first-flow evidence is visually accepted after this pass.

## Reason

XR-11.3 corrected the direction of the composer and removed part of the lens-selector ambiguity, but the product still read too much like a long form. The first screen did not reach the essential path fast enough, especially on mobile.

The product path must be obvious without reading repeated support copy:

```text
Topic → Lens → Copy Prompt
```

## Scope

XR-11.4 compresses the command flow without changing runtime contracts.

Implemented structural targets:

1. Convert the first composer into a stricter command-flow grid.
2. Hide the duplicate task header inside the composer.
3. Pull the Analysis Lens selector above secondary settings on mobile.
4. Keep the lens selector as explicit lens cards rather than a segmented empty-looking control.
5. Preserve two readable lens choices on narrow mobile instead of collapsing them into a long vertical form.
6. Hide or shrink AI motif pressure on mobile so it does not consume the first screen.
7. Move the copy prompt action closer to topic/lens selection through CSS ordering.
8. Hide Simple Mode advanced controls on narrow mobile.
9. Keep Expert Mode controls available.
10. Preserve all IDs, export contracts, RTL/mobile behavior, and the XR-8 evidence matrix.

## Product decisions

| Feature | XR-11.4 decision | Reason |
|---|---|---|
| Header / command bar | Compress on mobile header | It remains useful but must not dominate the first viewport. |
| AI motif | Hide on mobile | Decorative value is lower than the space cost. |
| Topic input | Keep and compress | It is the first true user action. |
| Analysis Lens selector | Rebuild as explicit lens cards | The user must clearly see Strategic and Biopolitical as choices. |
| Copy prompt action | Move closer to topic/lens path | Primary action should follow immediately after the core decision. |
| Advanced options | Hide in Simple Mode mobile | Secondary settings must not block the beginner path. |
| Mini guide / workflow strip | Hide in mobile composer | They duplicate the same first-step guidance. |
| Expert Mode | Keep power controls | Expert users still need full control surfaces. |

## Acceptance criteria

The next visual evidence audit must verify:

- `desktop-first-screen.png` no longer shows an ambiguous empty lens control.
- `mobile-first-screen.png` reaches topic and lens direction faster.
- `mobile-first-screen-dark.png` keeps the same compact command path.
- Simple Mode feels like a short guided product path.
- Expert Mode remains powerful but cleaner.
- The copy prompt action is visibly close to the first-flow decision.
- RTL/Arabic mobile does not overflow or clip controls.
- Browser core 50/50 passes.
- Hosted evidence and stable readiness pass.
- No-browser gates pass.

## Explicit non-scope

- No schema changes.
- No provider/backend/storage changes.
- No export contract changes.
- No OAuth or remote artifact changes.
- No JavaScript runtime changes.

## Next decision

If XR-11.4 evidence is visually accepted, resume RC-1.

If the first screen is still not satisfying, the next intervention must be a deeper markup-level product rebuild, not another CSS-only compression pass.
