# XR-11.4.2 — Advanced Controls Visibility Contract Hotfix

## Status

Hotfix package prepared after XR-11.4.1 browser validation still failed.

## Trigger

XR-11.4 compressed the mobile-first command flow and XR-11.4.1 restored the browser-tested expanded biopolitical prompt sample. The remaining systems-map failures came from `#promptMode` being hidden inside visually collapsed advanced controls.

The failing browser tests loaded the expanded biopolitical sample and then selected `research` from `#promptMode` before opening the prompt preview. Playwright could resolve the element, but it was not visible or enabled enough for interaction.

## Decision

Keep **Topic → Lens → Copy Prompt** compression, but do not hard-hide browser-tested advanced controls.

## Changed behavior

- Simple Mode advanced controls remain compact and secondary.
- `#promptMode` remains visible and enabled for browser contracts.
- `#preferredAssistant` and `#timeframeInput` remain visible with the same contract.
- The details grid is restored as a compact grid rather than hidden.
- Mobile uses a one-column compact advanced-control stack.
- The expanded biopolitical sample remains visible from XR-11.4.1.

## Preserved contracts

- runtime IDs unchanged
- export metadata unchanged
- lens import behavior unchanged
- RTL/mobile no-overflow behavior unchanged
- XR-8 evidence matrix unchanged
- systems-map browser tests preserved
- Simple Mode and Expert Mode remain structurally intact

## Acceptance criteria

## Release state

RC-1 remains blocked until the post-hotfix evidence archive is visually audited.
