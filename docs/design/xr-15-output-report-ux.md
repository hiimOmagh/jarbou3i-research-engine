# XR-15 — Biopolitical Output Report Recomposition

## Purpose

XR-15 converts the biopolitical export from a long theory stack into a readable report structure:

1. **short report first**
2. **expert appendix second**
3. **Evidence-capped confidence** before final judgment
4. strongest objection and weakest point before closure
5. lower-harm alternative before final recommendation

This follows the **Jarbou3i Life-Governance Analysis Engine**, which requires evidence caps, alternative hypotheses, disconfirmation conditions, prediction ledger discipline, and lower-harm alternatives before any model-grade final judgment.

## UX correction

The previous biopolitical cockpit gates were useful, but the exported report could still overload the user if every ontology layer appeared before the readable conclusion.

XR-15 makes the report user-friendly by separating:

- **Short report**: event, target population, dominant mechanism, evidence cap, final judgment, lower-harm alternative.
- **Expert appendix**: full Life-Governance apparatus, ontology, evidence matrix, red-team review, prediction ledger, disconfirmation conditions, contradiction heatmap, and lower-harm alternative details.

## Required report logic

The output must show:

- evidence-capped judgment
- dominant mechanism
- target population
- strongest evidence
- strongest objection
- weakest point
- conditional prediction
- watch indicators
- disconfirmation conditions
- lower-harm alternative
- trade-off
- residual risk

## Mode mapping

- **Simple Mode** sees the short report first.
- **Expert Mode** can inspect the expert appendix.
- **Evidence Mode** explains the confidence cap.
- **Final Mode** connects judgment, prediction, disconfirmation, and lower-harm alternative.

## Acceptance criteria

- The report begins with a readable summary, not a full theoretical dump.
- The expert appendix remains available and auditable.
- Evidence-capped confidence is visible before final judgment.
- The strongest objection and weakest point are visible before closure.
- The lower-harm alternative is not optional.
- Export contracts, runtime IDs, RTL/mobile behavior, and XR-8 evidence matrix remain preserved.

## Static contract terms

Biopolitical Output Report Recomposition; short report first; expert appendix second; Evidence-capped confidence; strongest objection; weakest point; lower-harm alternative; prediction ledger; disconfirmation conditions; Life-Governance apparatus; Evidence Mode; Final Mode; Jarbou3i Life-Governance Analysis Engine.
