# XR-12.1 — Integrated Cockpit Review System

## Status

RC-1 remains blocked until the cockpit migration is structurally integrated and visually re-audited.

XR-12.1 is a controlled architecture batch. It builds on XR-12.0 but corrects the main architectural weakness: the cockpit must not feel appended after the old workbench. It must become the Expert Mode review system.

## Product decision

Jarbou3i keeps two different user experiences:

- Simple Mode: guided Topic → Lens → Copy Prompt → Import path.
- Expert Mode: integrated Source → Structure → Brief cockpit.

The cockpit is not the landing page for everyone. It is the analysis/review engine for expert and imported-answer inspection.

## What changes

1. The Expert Cockpit becomes the dominant review surface in Expert Mode.
2. The old Expert Analyst Dashboard is demoted so it no longer competes with the new cockpit.
3. The Analysis Engine Map becomes a supporting structural module instead of the primary expert surface.
4. The Source panel now has meaningful empty state, pasted-input state, loaded state, schema state, repair state, model mode, source count, and lens metadata.
5. The Structure panel now exposes layer coverage, missing sections, flagged material, systems-map / evidence-pressure summary, and direct jumps to layer/evidence/scenario review.
6. The Brief panel now exposes an editorial preview, ranked findings, contradiction callout, and export/evidence actions.
7. Simple Mode continues to hide the dense cockpit and keeps the guided workflow intact.

## Source → Structure → Brief

### Source panel

The Source panel is responsible for explaining what has been imported or what still needs to be imported.

States:

- empty state: no structured answer pasted yet.
- pasted state: input exists but has not activated the cockpit.
- repair state: pasted content is not usable as a valid model response.
- loaded state: valid model response is imported and inspected.

### Structure panel

The Structure panel is responsible for model coverage and parser state.

It shows:

- layer coverage across the six model layers.
- item counts per layer.
- flagged contradiction/assumption pressure.
- supporting Analysis Engine Map access.
- systems-map summary for biopolitical imports.

### Brief panel

The Brief panel is responsible for the editorial intelligence preview.

It shows:

- thesis / title.
- top findings.
- confidence labels.
- first contradiction callout when present.
- export and evidence review actions.

## Relationship to existing modules

The old Expert Analyst Dashboard remains in the codebase for compatibility and rollback, but it is visually demoted in Expert Mode by the integrated cockpit.

The Analysis Engine Map remains available because it is useful, but it is no longer the dominant expert surface. It supports the Structure panel.

The Review panel and export contracts remain unchanged. Cockpit buttons jump into existing review/export surfaces instead of replacing them.

## Constraints preserved

- runtime IDs are preserved.
- export contracts are preserved.
- prompt sample visibility is preserved.
- advanced controls remain browser-visible.
- RTL and mobile behavior remain protected.
- XR-8 evidence matrix remains unchanged.
- no provider, backend, OAuth, storage, or schema migration is included.

## Acceptance criteria

- Expert Mode cockpit appears early and dominates review.
- Old Expert Analyst Dashboard no longer visually competes with the cockpit.
- Source / Structure / Brief panels are meaningful in empty and loaded state.
- Analysis Engine Map is structurally integrated as support, not duplicate architecture.
- Simple Mode still works as guided entry.
- Browser core 50/50 passes.
- Hosted evidence and stable readiness pass.
- No-browser passes.
