# XR-12.2 — Cockpit Loaded-State Editorial Depth

## Status

Implementation package for the integrated cockpit review system.

RC-1 remains blocked until the cockpit loaded state is visually and structurally strong enough for a release candidate. This milestone deepens the loaded Expert Mode cockpit without changing the schema, provider behavior, export contract, or storage model.

## Objective

Turn the Expert Cockpit from a useful shell into a meaningful loaded-state review surface. The target is a stronger `Source → Structure → Brief` experience after a valid AI structured answer is imported.

## Scope

### Source panel metadata

The Source panel now exposes a richer loaded-state source digest:

- schema health
- imported model route
- imported language
- word count
- source count
- contradiction count
- source completeness meter
- evidence-mapped diagnostic
- repair-needed versus review-ready state

This keeps the source digest useful before the user opens the full review panel.

### Structure panel diagnostics

The Structure panel now exposes parser pressure and layer health:

- total parsed items
- six-layer coverage
- missing layers
- flagged material
- evidence pressure
- systems coverage for the biopolitical lens
- quick links into engine, evidence, and scenario review tabs

The goal is to make parser pressure visible directly inside the cockpit rather than burying it in the older review dashboard.

### Brief panel editorial depth

The Brief panel now behaves more like an editorial intelligence preview:

- editorial thesis area
- brief readiness meter
- ranked preview findings
- contradiction signal callout
- next actions block
- export/evidence/contradiction routing

The Brief panel editorial depth should make the cockpit feel closer to the desired intelligence-desk direction while preserving the existing HTML export flow.

## Non-goals

- No schema changes.
- No backend/provider/storage/OAuth changes.
- No export contract changes.
- No removal of browser-tested runtime IDs.
- No changes to Simple Mode behavior.
- No replacement of the XR-8 evidence matrix.

## Contracts preserved

The implementation must preserve:

- runtime IDs
- export contracts
- import behavior
- prompt sample visibility
- advanced control visibility
- RTL/mobile no-overflow behavior
- XR-8 evidence matrix
- hosted evidence archive identity

## Static contract markers

The package introduces and guards these markers:

- `data-cockpit-depth="source-metadata"`
- `data-cockpit-source-digest="metadata-repair-state"`
- `data-cockpit-depth="structure-diagnostics"`
- `data-cockpit-parser-diagnostics="layer-pressure-evidence-pressure"`
- `data-cockpit-depth="editorial-brief-depth"`
- `data-cockpit-editorial-findings="ranked-preview"`
- `data-cockpit-next-actions="export-evidence-repair"`

## Acceptance criteria

- Browser core remains 50/50 green.
- Hosted evidence capture remains green.
- Stable release readiness remains green.
- No-browser checks remain green.
- The cockpit loaded state shows meaningful source digest, parser pressure, and brief readiness information.
- The Source panel metadata is useful without opening the full review panel.
- The Structure panel diagnostics make missing layers and evidence pressure visible.
- The Brief panel editorial depth feels like a stronger review/briefing surface.
- RC-1 remains blocked until visual audit confirms the cockpit direction is release-candidate acceptable.
