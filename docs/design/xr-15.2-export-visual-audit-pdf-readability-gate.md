# XR-15.2 — Export Report Visual Audit + PDF Readability Gate

## Purpose

XR-15.2 adds an export visual audit and PDF readability gate to the biopolitical report flow.

The goal is not another theory layer. The goal is to verify that the exported report reads like a professional intelligence document rather than a long web dump.

The gate preserves the intended report hierarchy:

1. short report first
2. evidence appendix next
3. expert appendix second

## Problem corrected

XR-15 and XR-15.1 improved report logic and print-safe evidence appendices, but the export still needs an explicit readability check before it can be treated as a serious PDF artifact.

Without this gate, a technically complete report can still fail as a document because it may have:

- weak page breaks
- excessive visual clutter
- dense stacked cards
- tables that are hard to scan
- buried final judgment
- appendix material that feels like raw HTML output
- no clear print-ready verdict

## New UX surface

XR-15.2 adds a biopolitical PDF readability panel with:

- export visual audit
- PDF readability gate
- page-break verification
- controlled report density
- evidence appendix readability check
- professional intelligence document verdict

## New export section

The exported biopolitical HTML report now includes an XR-15.2 visual audit gate with these checks:

- short report before appendices
- evidence appendix page-safe
- density controlled
- claim-source-gap table present
- professional intelligence document verdict

The section is print-safe and carries machine-readable contracts for browser and static validation.

## Readability standard

A report passes the XR-15.2 intent only if the reader can scan it in this order:

1. title and context
2. final judgment
3. dominant mechanism
4. confidence cap
5. strongest objection
6. evidence appendix
7. disconfirmation conditions
8. watch indicators
9. lower-harm alternative
10. trade-off and residual risk
11. expert appendix

## Design rule

Depth must not become visual clutter.

The report can be theoretically rich, but the PDF must still be legible, page-safe, and structured like a professional intelligence document.

## Life-Governance Engine alignment

XR-15.2 follows the Jarbou3i Life-Governance Analysis Engine requirement that analysis move from interpretation to mechanism, evidence, alternatives, prediction, disconfirmation, calibration, and lower-harm recommendations.

The added visual audit ensures this logic survives the final export surface instead of being lost in a long, unscannable HTML document.
