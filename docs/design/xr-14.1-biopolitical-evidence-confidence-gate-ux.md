# XR-14.1 — Biopolitical Evidence + Confidence Gate UX

## Status

Design/package milestone for making biopolitical analysis safer, clearer, and less overconfident.

XR-14 converted the Jarbou3i Life-Governance Analysis Engine into a question-first workflow. XR-14.1 adds the next user-friendly gate: **evidence before confidence, alternatives before final certainty, disconfirmation before judgment**.

## Why this exists

A biopolitical analysis can sound persuasive while resting on weak evidence. This is dangerous because the model analyzes life, bodies, populations, risk, care, exposure, market capture, and institutional residue.

The user should never see a confident conclusion before seeing:

- what evidence supports the claim;
- how reliable that evidence is;
- what the maximum confidence cap is;
- what alternative hypotheses remain plausible;
- what the strongest objection is;
- what would weaken or disprove the judgment.

## Evidence Confidence Score

The Evidence Confidence Score remains separate from LGI and RQS.

- LGI measures life-governance intensity.
- ECS measures evidence strength.
- RQS measures reasoning quality.

**No final confidence can exceed ECS.**

LGI intensity is not evidence confidence. A case can look structurally intense while remaining weakly supported.

## Confidence caps

The UI and prompt contract preserve the confidence caps from the Jarbou3i Life-Governance Analysis Engine:

| Condition | Maximum confidence |
|---|---:|
| mostly narrative evidence ≤40 | 40 |
| one source only ≤45 | 45 |
| no institutional evidence ≤60 | 60 |
| no direct evidence ≤65 | 65 |
| no alternative hypotheses ≤70 | 70 |
| no disconfirmation conditions ≤75 | 75 |
| no outcome evidence ≤80 | 80 |

These confidence caps are not decorative. They prevent theory, rhetoric, and visual confidence from outrunning evidence.

## Four visible gates

XR-14.1 adds a four-gate evidence UX for biopolitical analysis:

1. **Evidence class** — law, regulation, budget, procurement, contract, institutional action, database, infrastructure, corporate partnership, statistics, expert report, investigation, public speech, media narrative, anonymous claim, or interpretation.
2. **Evidence reliability** — authority, independence, directness, specificity, consistency, freshness, bias risk, verifiability, missing evidence, and chain of custody.
3. **Confidence cap** — the maximum allowed certainty after evidence limitations are applied.
4. **Alternatives + disconfirmation** — the strongest competing explanation and the conditions that weaken or disprove the conclusion.

## Product behavior

### Simple Mode

Simple Mode keeps the evidence gate readable. It does not expose every scoring table by default. It shows four checks and a small snapshot so a non-expert can understand why confidence is not automatic.

### Expert Mode

Expert Mode can expose the deeper audit language: Evidence Confidence Score, confidence caps, alternative hypotheses, strongest objection, disconfirmation, source class, and reliability dimensions.

### Evidence Mode

Evidence Mode becomes the bridge between the seven questions and the final judgment. It forces evidence classification before confidence.

### Final Mode

Final Mode must not output a clean final answer until it has considered:

- strongest evidence;
- weakest point;
- alternative hypotheses;
- strongest objection;
- disconfirmation;
- confidence cap;
- lower-harm alternatives.

## UX rule

The user-facing question is:

> Can this claim be trusted enough to support the final judgment?

The expert label is secondary:

> Evidence Confidence Score / confidence cap / disconfirmation.

## Non-goals

This package does not change provider execution, backend behavior, storage, OAuth, schema version, release identity, or export identity.

## Acceptance

XR-14.1 is acceptable only if:

- evidence appears before confidence in the biopolitical UX;
- confidence caps are visible and prompt-enforced;
- alternative hypotheses remain required;
- strongest objection and disconfirmation are required before final judgment;
- Simple Mode remains understandable;
- Expert Mode keeps the audit depth;
- Final Mode still includes lower-harm alternatives;
- the strategic lens remains unchanged;
- browser-tested controls remain visible and accessible;
- the XR-8 evidence matrix remains compatible.

## Source alignment

This milestone operationalizes the evidence protocol, confidence cap system, alternative hypothesis engine, red-team discipline, and lower-harm alternatives from the **Jarbou3i Life-Governance Analysis Engine** without exposing the full theory stack at once.
