# XR-15.1 — Export Report Evidence Appendix + Print Polish

## Purpose

XR-15.1 extends the biopolitical output report with a **print-ready appendix** that keeps the short report first and the expert appendix second, while making the exported report auditable on paper or PDF.

The goal is not to add another theory dump. The goal is to make the Jarbou3i Life-Governance Analysis Engine readable, printable, and audit-grade after the user has already seen the short judgment.

## UX rule

The report order remains:

1. Short report first.
2. Evidence appendix.
3. Expert appendix second.

The short report answers the user quickly. The appendix preserves evidence, uncertainty, objections, and alternatives.

## Required appendix sections

- Evidence matrix with **audit-grade evidence**.
- Claim/source/gap table using the **claim-source-gap** contract.
- Visible **confidence cap** before final judgment closure.
- **strongest objection** before conclusion protection.
- **disconfirmation conditions** and watch indicators.
- **lower-harm alternative** with trade-off and residual risk.
- Print-safe hierarchy and page behavior.

## Print polish requirements

- Use a **print-safe** appendix layout.
- Avoid breaking cards across pages.
- Keep heading hierarchy readable without the web UI shell.
- Preserve evidence rows in a compact table.
- Keep the reader oriented: report judgment first, apparatus later.

## Why this matters

The Life-Governance model requires evidence-ranked, adversarial, disconfirmable output. A report that looks impressive but hides evidence quality is not model-grade.

XR-15.1 therefore keeps the final export useful for:

- human reading
- PDF export
- print review
- audit trail
- expert appendix inspection
- evidence confidence review
