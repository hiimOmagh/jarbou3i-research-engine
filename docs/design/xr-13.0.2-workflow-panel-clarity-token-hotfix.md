# XR-13.0.2 — Workflow Panel Clarity Token Hotfix

## Purpose
Fix the remaining XR-13 no-browser static gate failure without changing runtime behavior or the user-friendly recomposition model.

The browser suite passed after XR-13.0.1, but the no-browser static gate still failed because the first-screen clarity contract expected the literal token `id="workflowPanel" data-clarity="workflow-start"`. The workflow panel already had the correct clarity state, but the attribute order did not match the literal static contract.

## Scope
- Reorder the `workflowPanel` attributes so the static contract token is present exactly.
- Preserve the compact visible mission anchor restored in XR-13.0.1.
- Preserve product-state separation and duplicate-guidance reduction from XR-13.
- Do not touch runtime logic, styling, import/export contracts, browser selectors, schema, or evidence behavior.
