# XR-11.1 — Mobile Hero Stack Repair

## Decision

XR-11 improved the product art direction and kept all evidence gates green, but the manual visual audit rejected one concrete mobile layout defect: the Arabic mobile first screen still inherited a two-column hero grid from the desktop composition.

That grid compressed the hero title into a narrow compressed rail beside the structured-intelligence signal. The result was technically valid but visually unacceptable for `mobile-first-screen.png` and risky for `mobile-first-screen-dark.png`.

## Scope

XR-11.1 is a targeted visual hotfix only.

It repairs:

- Arabic mobile hero title compression.
- The two-column hero grid on narrow mobile screens.
- The structured-intelligence signal placement.
- Mobile first-screen readability in light and dark evidence captures.

It does not change:

- Runtime IDs.
- Export contracts.
- JSON import behavior.
- Lens behavior.
- Provider/backend/schema/OAuth/storage behavior.
- XR-8 evidence matrix structure.

## Implementation

The mobile composer intro is forced into a one-column stack at `max-width:720px`.

The structured-intelligence signal moves above the hero copy. The title and explanatory text become centered, bounded, and readable instead of being squeezed into a side rail.

For Arabic RTL mobile specifically, the title receives a stricter width and line-height guard to prevent rail-like wrapping while preserving a compact first-screen appearance.

## Evidence targets

The repair must be verified against:

- `mobile-first-screen.png`
- `mobile-first-screen-dark.png`
- `desktop-first-screen.png`
- `desktop-first-screen-dark.png`
- `visual-evidence-matrix.json`

## Acceptance criteria

- Arabic mobile hero title is readable and not vertically compressed.
- The AI signal no longer steals horizontal width from the hero text on mobile.
- Light and dark mobile first screens are visually acceptable.
- RTL mobile overflow remains green.
- Browser core remains 50/50 green.
- Hosted evidence remains 2/2 green.
- Stable readiness remains green.
- No-browser remains green.

## Next step

If XR-11.1 passes the visual audit, return to `RC-1` and recreate the release-candidate checklist from the repaired visual state.
