# XR-11.4.1 — Prompt Sample Visibility Contract Hotfix

## Status

Hotfix package prepared after XR-11.4 browser validation failed.

## Trigger

XR-11.4 compressed the first-flow surface around **Topic → Lens → Copy Prompt**, but the compression also hid the `miniGuide` mini guide container that owns the browser-tested expanded biopolitical prompt sample.

The failing browser tests expected the card below to remain visible after selecting the biopolitical lens:

```text
[data-prompt-sample="expanded-biopolitical"]
```

The card still existed in the DOM, but it was hidden through an ancestor container. This broke the systems-map browser tests without breaking export, lens metadata, or no-browser checks.

## Decision

Keep the XR-11.4 first-flow compression, but preserve the expanded biopolitical sample visibility contract.

## Changed behavior

- The `miniGuide` mini guide container remains visible in compact form.
- The expanded biopolitical sample card is forced visible for the systems-map browser tests.
- The card is visually compressed, not restored as a large guidance block.
- On mobile, the sample card becomes a one-column compact surface.
- Descriptive paragraphs remain hidden where needed to preserve first-flow density.

## Preserved contracts

- runtime IDs unchanged
- lens import behavior unchanged
- export metadata unchanged
- RTL/mobile no-overflow behavior unchanged
- XR-8 evidence matrix unchanged
- Simple Mode and Expert Mode remain structurally intact

## Acceptance criteria

```text
systems-map prompt sample tests pass
expanded biopolitical sample is visible after lens selection
Topic → Lens → Copy Prompt compression remains intact
browser core 50/50 passes
hosted evidence passes
stable readiness passes
no-browser passes
```

## Release state

RC-1 remains blocked until the post-hotfix evidence archive is visually audited.
