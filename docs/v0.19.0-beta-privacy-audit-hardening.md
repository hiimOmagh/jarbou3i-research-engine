# v0.20.0-beta — Privacy Audit Hardening

This release upgrades privacy from basic export sanitation to an explicit release gate.

## Added

- `src/research/privacy-audit.js`
- `tests/privacy-audit-check.mjs`
- `tests/privacy-release-gate-check.mjs`
- `fixtures/privacy/browser-generated-export-v0.20.json`
- `docs/privacy-audit.md`
- `tests/v020-no-browser-suite.mjs`

## Behavior

The export path now routes through:

```text
payload → privacy export guard → privacy audit final pass → JSON export
```

The final pass removes sensitive keys and redacts secret-shaped text. The report preserves audit metadata without preserving secret values.

## Acceptance

- Exported JSON contains no raw provider API key.
- Exported JSON contains no raw OAuth access token.
- Exported JSON contains no raw OAuth refresh token.
- Exported JSON contains no bearer token string.
- Safe derived metadata like `token_hash` remains allowed.
- `privacy_export.release_gate` must be `pass`.
- `privacy_export.post_redaction_issue_count` must be `0`.
