# Evidence Artifact Review — v1.3.0-bio-rc.1

This release adds a deterministic review gate for hosted-demo evidence generated by the browser suite.

The review gate validates the evidence directory after `tests/hosted-demo-evidence.spec.js` runs. It does not create screenshots itself; it verifies that the generated evidence is complete, version-aligned, and semantically consistent with the public UI lock contract.

## Command

After running hosted evidence capture locally:

```bash
npm run test:browser:hosted
node tests/hosted-demo-evidence-review-check.mjs hosted-demo-evidence-local
```

or, using the package alias:

```bash
npm run test:evidence:hosted
```

The package alias reviews `hosted-demo-evidence-local/` by default. To review a GitHub artifact, unzip the artifact folder and pass its directory explicitly:

```bash
node tests/hosted-demo-evidence-review-check.mjs hosted-demo-evidence
```

## Required files

The review requires:

```text
desktop-first-screen.png
mobile-first-screen.png
strategic-mode.png
biopolitical-mode.png
visible-text-ar.json
visible-text-en.json
visible-text-fr.json
hosted-demo-metadata.json
```

## Metadata contract

`hosted-demo-metadata.json` must include:

```text
app_version = 1.3.0-bio-rc.1
evidence_version = 1.3.0-bio-rc.1
capture_set = public-ui-lock
generated_by = tests/hosted-demo-evidence.spec.js
projects include chromium and mobile-chrome
required_files includes every required evidence file
```

The public UI contract must confirm:

```text
app_version_meta = true
strategic_toggle = true
biopolitical_toggle = true
trilingual_visible_text = true
rtl_arabic = true
ltr_english_french = true
```

## Visible-text contract

Each visible-text snapshot must confirm:

```text
visible-text-ar.json -> html_lang ar, html_dir rtl
visible-text-en.json -> html_lang en, html_dir ltr
visible-text-fr.json -> html_lang fr, html_dir ltr
```

Each snapshot must also confirm Strategic and Biopolitical lens button visibility and contain substantial visible UI text.

## Release-candidate freeze rule

Alpha.7 is a release-candidate freeze milestone. It should not add product features, schema changes, UI redesigns, provider behavior, backend behavior, or new analysis semantics. Its purpose is to verify and freeze the evidence artifact chain around the already-green alpha.6 public UI.
