# v1.0.4 — Browser QA + Visual Regression Hardening

## Scope

v1.0.4 is a patch-only browser quality release. It adds no research-engine capability, no new provider/source/OAuth behavior, and no schema-breaking product feature.

## What changed

- Added browser layout persistence tests.
- Added viewport overflow checks for desktop, tablet, and mobile.
- Added visual regression capture scaffolding.
- Added optional strict visual baseline mode via `VISUAL_BASELINE_STRICT=1`.
- Added command-center and engine-map collapsed-state persistence.
- Added browser QA hardening metadata to exported packets.
- Added CI browser script coverage for provider, layout, visual capture, and full browser suite.

## New browser gates

```bash
npm run test:browser:provider
npm run test:browser:layout
npm run test:browser:visual
npm run test:browser:visual:strict
```

Default visual mode captures and attaches screenshots. Strict mode uses Playwright screenshot baselines and should be enabled only after approved baselines exist.

## Acceptance criteria

- No horizontal overflow on desktop, tablet, or mobile.
- Workflow tabs persist after reload.
- Command Center and Engine Map collapsed states persist after reload.
- Primary actions remain visible on first screen.
- Visual screenshots are attached to CI artifacts by default.
- Strict screenshot baseline mode is available for future locking.

## Non-goals

- No new UI feature.
- No new connector.
- No new provider mode.
- No production OAuth change.
