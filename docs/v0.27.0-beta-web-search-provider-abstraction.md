# v0.29.0-rc.1 — Web Search Provider Abstraction

## Purpose

v0.29 introduces a provider-neutral web-search abstraction without enabling live web search.

The release models the search layer that future compliant providers can implement:

- search provider identity
- query budget
- source diversity targets
- counter-evidence query requirements
- dry-run query plans
- source run ledger metadata
- Evidence Review Queue handoff rules

## Non-goals

This release does not add:

- live web search
- search-result scraping
- browser-side search API keys
- direct promotion of search results into the Evidence Matrix
- claims that dry-run plans verify external facts

## Search provider identities

The abstraction currently defines:

```text
mock_search
brave_search_api
tavily_search_api
serpapi
custom_search_api
```

Only `mock_search` is available as a dry-run fixture. Other providers are modeled as planned provider adapters.

## Safety contract

```text
web_search_api
→ dry-run query plan
→ source result ledger
→ Evidence Review Queue boundary
→ human review before Evidence Matrix promotion
```

Live search remains disabled:

```text
live_search_enabled:false
key_exported:false
raw_token_exported:false
review_gate:evidence_review_queue_required
```
