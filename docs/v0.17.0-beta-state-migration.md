# v0.17.0-beta — State Migration + Version Compatibility Layer

## Goal

Protect the research packet format from schema drift by adding an explicit migration layer before packet validation and import.

## Added

- `src/research/migrations.js`
- `packet_migration_report` in exported workflow packets
- migration fixtures for `v0.11.0-beta` through `v0.16.0-beta`
- `tests/migration-check.mjs`
- `tests/no-browser-qa-suite.mjs`
- `tests/v017-no-browser-suite.mjs`

## Behavior

Import now follows this order:

```text
raw packet → migrateResearchPacket() → validateWorkflowPacket() → state assignment → render/export
```

Migration defaults missing fields safely, disables live calls, disables remembered keys, renumbers evidence IDs, repairs causal-link evidence references, and attaches a report to the packet.

## Privacy rule

Migration is also a security boundary. Sensitive provider fields such as API keys, raw tokens, bearer values, secrets, and credential-like keys are redacted during migration. Exports still pass through the privacy export guard.

## Non-goals

- no real OAuth
- no real source crawling
- no provider capability expansion
- no UI refactor

This release is a stability layer, not a feature expansion.
