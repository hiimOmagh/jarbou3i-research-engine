# v0.27.0-beta — Real Source Connector Prototype

## Purpose

This release introduces the first controlled live source connector: GitHub public repository metadata.

The connector is deliberately narrow:

- public GitHub repository metadata only
- public releases metadata only
- public language metadata only
- backend-mediated fetches only
- no browser-side GitHub token
- no scraping
- no private repository access
- no automatic Evidence Matrix promotion

## Connector

```text
connector: github_public_repo
backend endpoint: /api/source-task
required input: owner/repo or https://github.com/owner/repo
```

The backend fetches:

```text
GET /repos/{owner}/{repo}
GET /repos/{owner}/{repo}/releases?per_page=N
GET /repos/{owner}/{repo}/languages
```

These are official GitHub REST API shapes. Public resources can be requested without user-side credentials; an optional server-side `GITHUB_TOKEN` may be used for deployment rate limits, but it is never exported.

## Review gate

Fetched source metadata becomes `evidence_candidates` and enters the Evidence Review Queue.

```text
GitHub public metadata → evidence candidates → Evidence Review Queue → human accept/edit/reject → Evidence Matrix
```

This release does not allow connector results to bypass human review.

## Safety contract

- `source_fetching_performed:true` is recorded only for the GitHub backend connector.
- `public_metadata_only:true` is returned by the Worker.
- `private_repo_supported:false` is returned by the Worker.
- `github_token_exposed:false` is returned by the Worker.
- Browser payload secrets are still stripped.
- Export Pack v2 and privacy audit remain active.

## Environment variables

```text
SOURCE_GITHUB_ENABLED=true

SOURCE_GITHUB_RELEASE_LIMIT=5
GITHUB_API_BASE_URL=https://api.github.com
GITHUB_API_VERSION=2022-11-28
GITHUB_TOKEN=<optional server-side token, never exported>
```
