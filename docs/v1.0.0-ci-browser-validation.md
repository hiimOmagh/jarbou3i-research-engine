# v1.0.0 CI / Browser Validation Status

## Status

v1.0.0 is still blocked in this sandbox.

The repository CI has been repaired so GitHub Actions can run the required release gates, but the local sandbox still cannot complete dependency installation. Therefore the full browser suite could not be executed here.

## CI repair completed

- Split CI into two jobs:
  - `no-browser` release gates
  - `browser` Playwright/provider E2E gates
- Added concurrency cancellation.
- Added explicit job timeouts.
- Added Playwright report/test-result artifact upload.
- Added shell-based CI scripts to avoid the Node aggregate child-process timeout pattern.
- Browser scripts now use the local project Playwright binary instead of accidentally resolving a global Python `playwright` command.
- Removed `wrangler` from `devDependencies`; deployment still uses `npx wrangler` from docs, but CI tests do not need the Wrangler package.

## New scripts

```bash
npm run test:ci:no-browser
npm run test:ci:browser
npm run test:ci
npm run test:release
```

## Local sandbox blocker

Dependency install timed out repeatedly:

```bash
npm install
npm install --package-lock-only --ignore-scripts --no-audit --no-fund
npm install @playwright/test@^1.56.0 --no-audit --no-fund --ignore-scripts
```

Because `node_modules` is unavailable, the browser gate now fails fast and correctly:

```text
Playwright Node CLI missing. Run npm install before browser CI.
```

## Browser gate required before v1.0.0

Run in GitHub Actions or a local machine with package installation access:

```bash
npm install --no-audit --no-fund
npm run test:ci:no-browser
npm run test:ci:browser
```

v1.0.0 must not be promoted until both CI jobs are green.
