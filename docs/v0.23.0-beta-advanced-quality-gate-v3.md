# v0.25.0-beta — Advanced Quality Gate v3

## Purpose

v0.23 turns the Quality Gate from a simple score panel into an actionable diagnostic layer. It does not add live integrations, real OAuth, or source fetching. It measures whether a research packet is structurally ready for review, publication, or further evidence work.

## Added

- `src/research/quality-gate.js` upgraded to Advanced Quality Gate v3.
- `quality_gate` is now exported in research workflow packets.
- `analysis_brief.quality_gate_report` is included in compiled briefs.
- Quality Gate UI now shows v3 dimensions, publication readiness, weakest dimensions, and fix actions.
- `tests/quality-gate-v3-check.mjs` validates the scoring layer and schema/fixture integration.
- `tests/v023-no-browser-suite.mjs` adds v0.23 release-gate coverage.

## Dimensions

Quality Gate v3 scores:

- completeness
- evidence strength
- contradiction coverage
- source diversity
- actor/layer coverage
- causal-link density
- provider safety
- privacy safety
- migration safety
- template fit

## Publication readiness labels

- `draft_only`
- `reviewable_draft`
- `research_ready`
- `publication_candidate`
- `blocked`

## Guardrails preserved

- Manual/private mode remains first-class.
- Evidence Review Queue remains mandatory for imported candidates.
- Provider output still requires validation before state mutation.
- Privacy export guard and privacy audit remain active.
- No real OAuth or live source fetching was added.

## Acceptance criteria

- Workflow schema requires `quality_gate`.
- Sample research workflow contains `quality_gate`.
- Compiled analysis brief contains `quality_gate_report`.
- v0.22 packets migrate to v0.23.
- Privacy/migration/provider/source tests continue to pass.
