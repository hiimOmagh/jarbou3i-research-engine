# v0.16.0-beta — Provider Mode Browser QA + Privacy Export Tests

## Purpose

v0.16.0-beta converts the v0.15 portable-account mock flow into a more testable provider layer. The release does not add real OAuth or live portable-account calls. It adds browser QA and export-time privacy enforcement.

## Implemented checks

- Provider selector browser coverage for `mock`, `openai_compatible`, `backend_proxy`, and `portable_oauth`.
- Portable OAuth mock connect/refresh/dry-run browser coverage.
- BYOK fake-key browser test confirming the key is never rendered into the UI.
- Privacy export guard that redacts sensitive fields before JSON download.
- Repository-level privacy audit for export, provider, portable, workflow, and fixture candidates.

## Privacy export policy

Allowed derived fields:

- `token_hash`
- `input_fingerprint`
- `prompt_fingerprint`
- `key_exported:false`
- `raw_token_exported:false`
- `access_token_exported:false`
- `refresh_token_exported:false`

Redacted fields/patterns:

- raw access tokens
- raw refresh tokens
- API keys
- bearer strings
- provider secrets
- client secrets
- private keys
- credential/password fields

## Commands

```bash
npm run test:privacy:guard
npm run test:privacy:export
npm run test:privacy
npm run test:browser:provider
npm run test:v016
```

## Boundary kept intentionally

This release does not implement real OAuth, real BrainLink/OpenRouter integration, live portable-account inference, or billing. It only prepares the UX/test/privacy envelope.
