# v0.20.0-beta — UX Reliability Pass

## Purpose

This release improves reliability at the user-interaction layer without adding new core analytical capabilities.

The goal is to make the research engine feel safer, clearer, and less brittle for real users while preserving the v0.19 privacy audit release gate.

## Added

- `src/research/ux-reliability.js`
- Provider mode guide rendered directly inside the Provider Harness.
- Rich empty states for major research panels.
- Toast-backed research status feedback.
- Button disabled states for unavailable actions.
- Destructive-action confirmations before import/reset-style operations.
- Export confirmation summary before research packet download.
- `tests/ux-reliability-check.mjs`
- `tests/v020-no-browser-suite.mjs`

## UX reliability changes

### Empty states

The main panels now explain what to do next instead of showing only a passive empty line.

Covered panels include:

- Research Plan
- Evidence Matrix
- Causal Links
- Analysis Brief
- Evidence Review Queue
- Provider Run Ledger

### Disabled states

Buttons that cannot currently perform useful work are disabled with a short reason.

Examples:

- Copy plan prompt is disabled before a plan exists.
- Clear evidence is disabled when the Evidence Matrix is empty.
- Export run ledger is disabled when no provider runs exist.
- Accept edited review candidate is disabled until a candidate is loaded for editing.

### Destructive-action confirmations

The app now warns before replacing or deleting meaningful workflow state.

Covered actions include:

- Import research packet
- Clear research plan
- Clear Evidence Matrix
- Clear causal links
- Clear analysis brief
- Clear provider run ledger
- Clear source import preview
- Clear resolved review items

### Export confirmation summary

Before exporting a research packet, the UI summarizes:

- topic
- evidence count
- causal-link count
- review-queue count
- pending review count
- provider-run count
- source-import count
- privacy gate state

The privacy audit/export guard still performs the actual export-time sanitization.

### Provider mode guide

The Provider Harness now explains the operational difference between:

- MockProvider
- OpenAI-compatible BYOK
- Hosted backend proxy
- Portable account mock

This reduces ambiguity around auth, billing, privacy, and production status.

## Guardrails preserved

- Manual/private mode remains first-class.
- Export privacy guard remains active.
- Privacy audit final pass remains active.
- No raw API key or OAuth token is exported.
- Portable OAuth remains mock-only.
- Source automation remains planning-only/review-gated.

## Acceptance criteria

- Existing no-browser QA passes.
- UX reliability helper test passes.
- Browser provider spec remains syntactically valid.
- No provider, source, migration, or privacy behavior regresses.
- No new live capability is introduced.
