# v1.0.5 — Onboarding + First-Run Success

This patch adds a local-only first-run guide that turns the first session into an explicit operational path:

```text
Topic → Plan → Evidence → Review queue → Quality gate → Safe export
```

## Added

- `src/research/onboarding.js`
- first-run guide panel in `index.html`
- local onboarding state in `state-store`
- onboarding report metadata in exported research packets
- schema support for `onboarding`
- canonical v1.0.5 migration/privacy fixtures
- `tests/onboarding-first-run-check.mjs`
- `tests/v105-no-browser-suite.mjs`

## Compatibility boundary

- Manual/private mode remains the default.
- No live provider behavior changes.
- No production OAuth change.
- No backend or connector behavior change.
- First-run state is local-only and export-safe.

## Acceptance criteria

- A new user can identify the next action without opening advanced panels.
- The first-run guide exposes the core workflow without hiding manual operation.
- Existing patch gates, browser QA hardening, privacy export checks, migration checks, and source/provider boundaries continue to pass.
