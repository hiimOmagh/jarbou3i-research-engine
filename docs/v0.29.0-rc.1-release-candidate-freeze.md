# v0.29.0-rc.1 — Release Candidate Freeze

## Purpose

This release freezes the feature surface before v1.0.0.

The release candidate is not a feature expansion. It adds release-gate metadata, a freeze policy, RC tests, and packaging documentation so the project can move from beta development into stabilization.

## Freeze policy

```text
feature_freeze: true
breaking_changes_allowed: false
production_oauth_allowed: false
new_live_connectors_allowed: false
export_privacy_regression_allowed: false
```

## Allowed work during RC

```text
bugfix
docs
qa
a11y
privacy_audit
migration_compatibility
release_packaging
```

## Blocked work during RC

```text
new_major_feature
new_live_provider
new_source_connector
oauth_production_enablement
schema_break_without_migration
secret_export_weakening
```

## Acceptance criteria

- The app exports `release_candidate` metadata.
- Schema requires `release_candidate` metadata.
- v0.28 packets migrate to `v0.29.0-rc.1`.
- OAuth token/export guardrails remain enforced.
- Web search remains dry-run unless later changed in a post-RC branch.
- Existing source connector results still route through the Evidence Review Queue.
- Browser provider QA remains the final local/CI gate before v1.
